# 商照管理后台 — 服务器部署完整指南

## 前置条件

| 项目 | 要求 |
|------|------|
| 操作系统 | Linux（Ubuntu/Debian 或 CentOS/RHEL） |
| Docker | 已安装 Docker + Docker Compose v2+ |
| 网络 | 服务器与 inSona 网关在同一网络（TCP 8091 可达） |
| Git | 已安装 git |

> Node.js **不需要**安装在宿主机（容器内自带）

---

## 方法一：一键部署（推荐）

```bash
# 克隆项目
git clone <仓库地址> insona-admin
cd insona-admin

# 运行部署脚本
bash scripts/deploy.sh
```

脚本会自动完成：环境检查、交互式配置、.env 生成、Docker 构建、启动、健康检查。

---

## 方法二：手动部署

### 步骤 1：克隆项目

```bash
git clone <仓库地址> insona-admin
cd insona-admin
```

### 步骤 2：配置环境变量

```bash
cp .env.example .env
nano .env
```

| 变量 | 说明 | 示例 |
|------|------|------|
| `DATABASE_URL` | SQLite 数据库路径（容器内） | `file:/app/data/dev.db` |
| `GATEWAY_IP` | inSona 网关 IP 地址 | `192.168.1.100` |
| `GATEWAY_PORT` | inSona 网关端口 | `8091` |

### 步骤 3：构建并启动

```bash
docker compose up -d --build
```

首次构建需 2-5 分钟。

### 步骤 4：验证部署

```bash
# 检查容器状态
docker compose ps

# 查看日志
docker compose logs -f

# 测试 HTTP
curl http://localhost:3000
```

浏览器访问 `http://<服务器IP>:3000`

## 开机自启动（systemd）

```bash
# 编辑 deploy/insona-admin.service，将 WorkingDirectory 改为实际部署路径
sudo cp deploy/insona-admin.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable insona-admin
sudo systemctl start insona-admin

# 验证
sudo systemctl status insona-admin
```

## 日志管理

### 应用日志

```bash
# Docker 模式
docker compose logs -f insona-admin

# systemd 模式
sudo journalctl -u insona-admin -f
```

### 能耗事件日志

能耗事件自动输出到 `./data/logs/energy.log`（JSON Lines 格式，每行一个事件）。

#### 配置日志轮转

```bash
sudo cp deploy/logrotate.conf /etc/logrotate.d/insona-admin
# 编辑 logrotate.conf 中的日志路径为实际路径
sudo nano /etc/logrotate.d/insona-admin
```

轮转策略：每日轮转，保留 30 天，自动压缩。

## 数据持久化与备份

SQLite 数据库存储在 `./data/dev.db`，通过 Docker volume 持久化。

```bash
# 备份
cp ./data/dev.db ./data/dev.db.backup.$(date +%Y%m%d)

# 恢复
cp ./data/dev.db.backup.20260415 ./data/dev.db
docker compose restart
```

## 常用运维操作

| 操作 | 命令 |
|------|------|
| 查看实时日志 | `docker compose logs -f` |
| 查看最近 100 行日志 | `docker compose logs --tail=100` |

| 查看网关连接状态 | `curl http://localhost:3000/api/gateway/status` |

## 故障排查

### 网关无法连接

```bash
# 从容器内测试网络连通性
docker compose exec insona-admin nc -zv <网关IP> 8091
```

确认网关 IP 正确，防火墙允许 8091 端口出站。

### 容器启动后立即退出

```bash
docker compose logs --tail=50
```

常见原因：Prisma 迁移失败、端口被占用、.env 配置错误。

### 端口冲突

修改 `docker-compose.yml` 中的端口映射：

```yaml
ports:
  - "3001:3000"
```

## 架构说明

| 组件 | 说明 |
|------|------|
| Next.js 15 | App Router，standalone 输出 |
| SQLite | 单文件数据库，volume 持久化 |
| inSona 网关 | TCP 8091，JSON 双向通信 |
| 后台调度器 | instrumentation hook 自动启动，每 60 秒检查定时任务 |
| 网关自动连接 | 应用启动时从 .env 读取 GATEWAY_IP/PORT 自动连接 |
| 断线重连 | 指数退避（1s~60s），最多 10 次 |
| 能耗日志 | JSON Lines 格式写入独立文件，每日轮转 |
