# 商照管理后台生产部署指南

本文档用于在 Linux 服务器上以生产模式部署当前项目，包含：

- `npm run start` 的 `systemd` 配置
- 生产环境部署步骤
- 从 `dev` 模式切到 `prod` 模式的检查清单

## 1. 部署前提

服务器需要满足以下条件：

- Linux 系统（Ubuntu / Debian / CentOS / Rocky Linux 均可）
- Node.js 20
- npm 可用
- Git 已安装
- 服务器可以访问 inSona 网关 TCP `8091`
- 项目目录和 SQLite 数据库目录对运行用户可写

建议先确认版本：

```bash
node -v
npm -v
git --version
```

## 2. 项目初始化

### 2.1 拉取代码

```bash
cd /home/abc
git clone <仓库地址> insona-admin
cd insona-admin
```

### 2.2 安装依赖

```bash
npm ci
```

### 2.3 配置环境变量

如果项目目录中没有 `.env`，先创建：

```bash
cp .env.example .env
```

建议最少包含以下内容：

```env
NODE_ENV=production
PORT=3000
DATABASE_URL="file:./prisma/dev.db"
```

说明：

- `NODE_ENV=production`：强制生产模式
- `PORT=3000`：应用监听端口
- `DATABASE_URL="file:./prisma/dev.db"`：SQLite 数据库路径

如果你使用的数据库路径不是 `./prisma/dev.db`，请确保目标目录可写。

### 2.4 修正权限

假设服务运行用户为 `abc`：

```bash
sudo chown -R abc:abc /home/abc/insona-admin
chmod 755 /home/abc/insona-admin
chmod 755 /home/abc/insona-admin/prisma
```

如果你不是 `abc` 用户，请替换为实际用户名。

## 3. Prisma 初始化

### 3.1 生成 Prisma Client

```bash
npx prisma generate
```

### 3.2 初始化数据库结构

```bash
npx prisma db push
```

说明：

- 如果 `prisma/dev.db` 不存在，`db push` 会自动创建
- 如果报 `readonly database`，说明数据库目录或项目目录没有写权限

## 4. 生产构建与手动验证

### 4.1 构建

```bash
npm run build
```

### 4.2 手动启动验证

先不要急着上 `systemd`，先验证应用能独立启动：

```bash
npm run start
```

然后新开一个终端测试：

```bash
curl http://127.0.0.1:3000/api/gateway/status
```

如果返回 JSON，说明基础服务已启动成功。

确认无误后停止手动启动进程，再进入 `systemd` 托管。

## 5. `systemd` 配置

### 5.1 创建 service 文件

创建：

```bash
sudo nano /etc/systemd/system/insona-admin.service
```

写入以下内容：

```ini
[Unit]
Description=inSona Admin Production Service
After=network.target

[Service]
Type=simple
User=abc
Group=abc
WorkingDirectory=/home/abc/insona-admin
Environment=NODE_ENV=production
Environment=PORT=3000
Environment=DATABASE_URL=file:./prisma/dev.db
ExecStart=/usr/bin/npm run start
Restart=always
RestartSec=5
KillMode=control-group
TimeoutStopSec=30

[Install]
WantedBy=multi-user.target
```

需要按实际情况修改：

- `User=abc`
- `Group=abc`
- `WorkingDirectory=/home/abc/insona-admin`
- `ExecStart=/usr/bin/npm run start`

如果不确定 `npm` 路径，先执行：

```bash
which npm
```

然后替换 `ExecStart` 中的路径。

### 5.2 启用服务

```bash
sudo systemctl daemon-reload
sudo systemctl enable insona-admin
sudo systemctl start insona-admin
```

### 5.3 查看状态

```bash
sudo systemctl status insona-admin
```

### 5.4 查看实时日志

```bash
sudo journalctl -u insona-admin -f
```

### 5.5 常用命令

```bash
sudo systemctl restart insona-admin
sudo systemctl stop insona-admin
sudo systemctl disable insona-admin
```

## 6. 更新代码后的标准流程

如果服务器已经运行过旧版本，更新代码推荐按以下顺序：

```bash
cd /home/abc/insona-admin
git pull
npm ci
npx prisma generate
npx prisma db push
npm run build
sudo systemctl restart insona-admin
```

如果依赖没有变化，也可以先试：

```bash
git pull
npx prisma generate
npm run build
sudo systemctl restart insona-admin
```

## 7. 从 `dev` 模式切到 `prod` 模式检查清单

### 7.1 环境检查

- Node.js 版本为 20.x
- `npm ci` 已执行
- `.env` 已存在
- `DATABASE_URL` 指向服务器本地可写路径
- `prisma/` 目录可写

### 7.2 Prisma 检查

- `npx prisma generate` 已执行
- `npx prisma db push` 已执行
- 不再出现 `Unknown argument ...`
- 不再出现 `attempt to write a readonly database`

### 7.3 构建检查

- `npm run build` 成功
- `.next` 构建产物已生成
- 启动时没有 Next.js 配置级阻断错误

### 7.4 运行检查

- `npm run start` 可以手动启动
- `curl http://127.0.0.1:3000/api/gateway/status` 返回正常
- 能新增网关
- 能连接网关
- 能控制单设备
- 能控制组设备

### 7.5 功能回归检查

- 多网关状态显示正常
- 首页状态正常
- 控制页状态正常
- 设备控制抽屉状态同步正常
- 定时任务可执行
- 后台调度工作正常

### 7.6 日志检查

当前项目已做日志收敛，默认不会打印过多底层噪音。

如需额外排查，可临时加环境变量：

```env
PRISMA_LOG_ERRORS=true
GATEWAY_DEBUG=true
GATEWAY_TRACE_RAW=true
```

说明：

- `PRISMA_LOG_ERRORS=true`：打开 Prisma 底层错误日志
- `GATEWAY_DEBUG=true`：打开网关普通调试日志
- `GATEWAY_TRACE_RAW=true`：打开网关原始报文日志

不建议长期在生产环境一直开启 `GATEWAY_TRACE_RAW=true`，会明显增加日志量和 I/O。

## 8. 故障排查

### 8.1 `next: not found`

说明依赖没有安装：

```bash
npm ci
```

### 8.2 `Unknown argument name`

说明 Prisma Client 没有按当前 schema 重新生成：

```bash
npx prisma generate
```

### 8.3 `readonly database`

说明数据库文件或目录没有写权限，修复：

```bash
sudo chown -R abc:abc /home/abc/insona-admin
chmod 755 /home/abc/insona-admin/prisma
```

然后重新执行：

```bash
npx prisma db push
```

### 8.4 添加网关失败但数据库文件不存在

优先检查 `.env` 中的 `DATABASE_URL` 是否指向了错误路径，推荐：

```env
DATABASE_URL="file:./prisma/dev.db"
```

然后执行：

```bash
npx prisma generate
npx prisma db push
```

### 8.5 网关连接正常但控制失败

先检查：

```bash
curl http://127.0.0.1:3000/api/gateway/status
```

再查看应用日志：

```bash
sudo journalctl -u insona-admin -f
```

## 9. 建议

正式环境建议始终使用：

```bash
npm run build
npm run start
```

不要长期使用：

```bash
npm run dev
```

原因：

- `dev` 模式资源占用更高
- 稳定性更差
- 更容易出现重复初始化、日志过量、上下文不一致等问题
- 对这种带 TCP 网关连接、SSE、后台调度的项目尤其不合适
