  <action>
Create `deploy/insona-admin.service` — a systemd unit file that manages docker compose lifecycle for auto-start on boot (PROCESS-03).

**deploy/insona-admin.service** must contain:
```
[Unit]
Description=inSona Admin - Commercial Lighting Control Panel
Requires=docker.service
After=docker.service

[Service]
Type=oneshot
RemainAfterExit=yes
WorkingDirectory=/opt/insona-admin
ExecStart=/usr/bin/docker compose up -d
ExecStop=/usr/bin/docker compose down
ExecStopPost=/usr/bin/docker compose rm -f

[Install]
WantedBy=multi-user.target
```

The WorkingDirectory `/opt/insona-admin` is the assumed deployment path (documented in DEPLOY.md). The `After=docker.service` ensures Docker is available before starting.

Then, add a healthcheck to `docker-compose.yml` under the `insona-admin` service:
```yaml
healthcheck:
  test: ["CMD", "wget", "--no-verbose", "--tries=1", "--spider", "http://localhost:3000"]
  interval: 30s
  timeout: 10s
  retries: 3
  start_period: 40s
```

This uses `wget` (available in alpine) to verify the HTTP endpoint is responding. The `start_period: 40s` accounts for Next.js cold start + Prisma migration time.
