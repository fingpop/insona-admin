    - scripts/deploy.sh exists and starts with #!/bin/bash shebang
    - Script checks for prerequisites: docker (docker --version), docker compose (docker compose version), git (git --version)
    - If prerequisites missing, script exits with error message and exit code 1
    - Script prompts for GATEWAY_IP with read -p, shows current network interfaces as hint
    - Script uses default GATEWAY_PORT=8091 but allows override
    - Script uses default DEPLOY_DIR=/opt/insona-admin but allows override
    - Script generates .env file with: DATABASE_URL=file:/app/data/dev.db, GATEWAY_IP=<user input>, GATEWAY_PORT=<user input or default>, LOG_PATH=/app/data/logs/energy.log
    - Script runs: git clone (if repo URL provided) OR uses current directory
    - Script creates data directory: mkdir -p ./data && chmod 755 ./data
    - Script creates log directory: mkdir -p ./data/logs
    - Script runs: docker compose up -d --build
    - Script waits for health check: polls docker compose ps until status shows "healthy" or timeout (120s)
    - Script verifies deployment: curl http://localhost:3000 and checks HTTP 200
    - Script outputs success message with URLs and next steps
    - On failure at any step, script outputs error and suggests troubleshooting (docker compose logs)
    - Script is idempotent: running on existing deployment skips clone, regenerates .env, rebuilds
  </acceptance_criteria>
  <action>
    Create scripts/deploy.sh as an interactive bash script that automates the entire deployment flow.

    Structure the script with these sections:

    1. **Header and setup**: shebang, set -e for error handling, color output variables (optional)

    2. **Prerequisites check**:
       ```bash
       command -v docker >/dev/null 2>&1 || { echo "Docker required"; exit 1; }
       docker compose version >/dev/null 2>&1 || { echo "Docker Compose required"; exit 1; }
       command -v git >/dev/null 2>&1 || { echo "Git required"; exit 1; }
       ```

    3. **Interactive configuration**:
       - Prompt for GATEWAY_IP: `read -p "Gateway IP (e.g., 192.168.1.100): " GATEWAY_IP`
       - Prompt for GATEWAY_PORT with default: `read -p "Gateway port [8091]: " GATEWAY_PORT; GATEWAY_PORT=${GATEWAY_PORT:-8091}`
       - Prompt for deploy directory: `read -p "Deploy directory [/opt/insona-admin]: " DEPLOY_DIR; DEPLOY_DIR=${DEPLOY_DIR:-/opt/insona-admin}`
       - Prompt for repo URL (optional): `read -p "Git repo URL (leave empty if already cloned): " REPO_URL`

    4. **Clone or navigate**:
       - If REPO_URL provided and directory doesn't exist: git clone $REPO_URL $DEPLOY_DIR
       - If directory exists, cd to it

    5. **Generate .env**:
       ```
       cat > .env << ENVEOF
       DATABASE_URL="file:/app/data/dev.db"
       GATEWAY_IP="$GATEWAY_IP"
       GATEWAY_PORT=$GATEWAY_PORT
       ENVEOF
       ```

    6. **Create directories**:
       ```
       mkdir -p ./data/logs
       chmod 755 ./data
       ```

    7. **Build and start**:
       ```
       docker compose up -d --build
       ```

    8. **Wait for healthy**:
       - Loop up to 120s, checking `docker compose ps` for "healthy" status
       - Or fallback: check `curl -s -o /dev/null -w "%{http_code}" http://localhost:3000` returns 200

    9. **Verify and report**:
       - Show container status
       - Show URL: http://<server-ip>:3000
       - Remind about logrotate: `sudo cp deploy/logrotate.conf /etc/logrotate.d/insona-admin`
       - Remind about systemd: `sudo cp deploy/insona-admin.service /etc/systemd/system/ && sudo systemctl enable insona-admin`

    10. **Error handling**: Use trap or check exit codes at each step, output helpful error messages

    What to avoid:
    - Do NOT hardcode the repo URL (let user provide it)
    - Do NOT skip the prerequisites check
    - Do NOT use sudo inside the script (user should run with sudo if needed)
    - Do NOT install Docker or other prerequisites (just check and exit if missing)
    - Do NOT modify existing .env without warning (check if exists, ask to overwrite)
  </action>
  <verify>
    <automated>head -1 scripts/deploy.sh | grep -q "#!/bin/bash" && grep -c "docker compose" scripts/deploy.sh && grep -c "GATEWAY_IP" scripts/deploy.sh</automated>
  </verify>
  <done>
    - scripts/deploy.sh is executable (chmod +x), has bash shebang, set -e
    - Script checks docker, docker compose, git prerequisites
    - Script prompts for GATEWAY_IP, GATEWAY_PORT (default 8091), DEPLOY_DIR (default /opt/insona-admin)
    - Script generates .env with DATABASE_URL, GATEWAY_IP, GATEWAY_PORT, LOG_PATH
    - Script creates ./data/logs directory
    - Script runs docker compose up -d --build
    - Script waits for and verifies healthy deployment
    - Script outputs success message with next steps (logrotate, systemd)
    - Script handles errors gracefully with helpful messages
  </done>
